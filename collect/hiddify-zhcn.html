四、如何在 HiddifyApp 中使用 UDP Turn Relay

如何使用 UDP Turn Relay

最近，一种名为 UDP Turn Relay 的功能被用于 Messenger 中以绕过过滤。也就是说，在这种情况下，Messenger 被用作中间服务器（中继），流量通过它们发送到主服务器。这可以用于在伊朗境内活跃并支持此协议的 Messenger，或用于未被过滤的外国 Messenger。在这种情况下免费网络连接的示意图如下。

![UDP 转中继](https://github.com/hiddify/hiddify-next/assets/125398461/037262d4-ad89-45c0-bcab-c6504c08d3df)

请注意，此功能仅适用于 UDP 协议上的配置。这意味着此功能可以在 Hysteria2 或 TUIC v5 上使用，但不能用于现实。

要使用它，只需按照以下命令将该 Messenger 的参数放在配置链接的末尾。 

relay=urlencoded(turn://user:pass@host:port?realm=t)

其中 user 是连接到所需 Messenger 中的协议的用户名。 pass 也是该 Messenger 中此帐户的密码。 此外， host 是连接到 Messenger 的主机名， port 是连接到 Messenger 的端口。 该 realm 参数与主机名内的身份验证区域相关。 也就是说，可以使用此帐户对 Messenger 的哪个区域进行身份验证。

例如，在Bale messenger（伊朗信使）中，这些参数如下。 

relay=urlencoded(turn://balelivekit:GygZPHQSgAV7L5L8@meet-turn.bale.sh:443?realm=bale.ai)

首先，使用编码器将此信息转换为 UTF-8。例如，您可以使用 此编码器 ，其输出如下： 

relay=turn%3A%2F%2Fbalelivekit%3AGygZPHQSgAV7L5L8%40meet-turn.bale.sh%3A443%3Frealm%3Dbale.ai

现在，如果我们有一个基于 UDP 的 hysteria 配置，如下所示： 

hysteria2://87543215-15ac-54f6-3895-34e4e931f5b4@45.120.34.46:11464?hiddify=1&obfs=salamander&obfs-password=bdgf!huy678559j89GrTu&sni=www.speedtest.net&insecure=1#test

最后，我们在 hysteria 链接的末尾添加与中继（信使）相关的编码信息。为此，我们首先放置一个 & 再在#前准确复制粘贴链接， 如下所示： 

hysteria2://87543215-15ac-54f6-3895-34e4e931f5b4@45.120.34.46:11464?hiddify=1&obfs=salamander&obfs-password=bdgf!huy678559j89GrTu&sni=www.speedtest.net&insecure=1&relay=turn%3A%2F%2Fbalelivekit%3AGygZPHQSgAV7L5L8%40meet-turn.bale.sh%3A443%3Frealm%3Dbale.ai#test

您可以在Hiddify-Next程序中导入此链接，并且由于该程序支持UDP Turn Relay协议，因此您可以使用此功能。

笔记！

Bale Messenger 在短时间内禁用了该功能。根据上述内容，您应该从 伊朗内部 Messenger 列表 或未过滤的外部 Messenger 中找到支持此功能的 Messenger。

Profile Title：

如果以下列表中的第一项不为空或为 null，我们将使用它

  • “Profile Title”标题
  • “content-disposition” 标题（如下所述）
  • URL 片段（URL 中 # 后面的任何内容）
  • URL 的最后一部分作为配置文件名。任何 .txt .json .yaml 或 .yml 将被删除

Response headers

Profile-Title

您可以使用 base64 编码的 UTF8 名称作为Profile-Title，这样您就可以在Profile-Title中使用表情符号😍但是您也可以在标题中使用 ascii 名称。例： 

Profile-Title: base64:SSDinaTvuI8gSGlkZGlmeQ==
它将把Profile-Title显示为 I ❤️ Hiddify 

Profile-Title: I love hiddify
它将把Profile-Title显示为 I love Hiddify

content-disposition

如果 Profile-Title 未设置以向后兼容冲突配置，我们将使用与作为配置文件名对应的值 filename ，否则使用 URL 的最后一部分作为配置文件名。任何 .txt 、 .yaml 或 .yml 都将被删除 

content-disposition: attachment; filename="abc.txt"

配置文件更新间隔

如果响应头中有 profile-update-interval 字段，则将配置文件自动更新间隔设置为相应值，单位为小时 

profile-update-interval: 12

订阅用户信息

如果 subscription-userinfo 响应头中存在该字段，其对应的流量和过期信息将会显示在Profiles模块中， 规范参考这里 

subscription-userinfo: upload=455727941; download=6174315083; totl=1073741824000; expire=1671815872

profile-web-page-url

如果 profile-web-page-url 响应头中存在该字段，则该 Open web page 选项将显示在配置文件中，允许用户跳转到相应的门户主页

support-url

如果 support-url 响应头中存在该字段，则该 support 选项将显示在配置文件中，允许用户跳转到相应的门户主页

moved-permanently-to

如果 moved-permanently-to 响应头中存在该字段，这将更新订阅网址。当您的子链接被阻止或您想要更改域时，它很有用。

DNS

如果 DNS 响应标头中存在该字段，并且用户未设置自己的首选 DNS，则应用程序将使用此 DNS 服务器。（在冲突配置文件中可以覆盖此设置）

fragment 参数

如果你想在链接中使用 Fragment，你必须考虑以下参数 

&fragment=size,sleep,hellotls

例如，你只需要在链接中输入以下信息

&fragment=10-100,100-200,hellotls

MUX 参数

如果你想在链接中设置 Mux 参数 

mux : "smux/h2mux/yamux"mux_max : "number" e.g. 6mux_min : "number" e.g. 2mux_pad : "true/false"mux_down : "number in mbps" e.g. 100mux_up : "number in mbps" e.g. 200

这里：

  • mux 可以是 smux 、 yamux 、 h2mux

  • mux_max 是最大连接数。

  • mux_min 是最小流数。

  • mux_pad 是打开或关闭 MUX 上的此功能的 true 或 false 参数。它应该与服务器相同。

  • mux_down 是 MUX 的下载带宽，单位为 mbps。

  • mux_up 是 MUX 的上传带宽（mbps）。

例子： mux=smux&mux_max=4&mux_min=0&mix_pad=false&mux_up=10&mux_down=10
